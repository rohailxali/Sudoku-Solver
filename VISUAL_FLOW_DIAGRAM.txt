█ SECTION 6: KEY ALGORITHM BEHAVIORS
═══════════════════════════════════════════════════════════════════════════════

AC-3 BEHAVIOR:
├─ Removes values that violate constraints
├─ Iterates until no more changes (fixed point)
├─ Complexity: O(n²d³)
├─ Very efficient for Sudoku (n=81, d=9)
└─ Often solves easy puzzles alone!

BACKTRACKING BEHAVIOR:
├─ Only explores necessary branches
├─ MRV heuristic reduces branching factor
├─ Forward checking catches conflicts early
├─ Exponential worst case, but well-pruned
└─ Typically < 50 iterations for any Sudoku

IMPACT OF MRV HEURISTIC:
├─ Without MRV: 9! × 8! × 7! × ... branches
│  (astronomical search space)
├─ With MRV: 2^k branches (where k << 81)
│  (manageable search space)
└─ Example impact:
   Without: Might take hours
   With: Milliseconds!
