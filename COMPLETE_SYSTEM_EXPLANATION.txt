🧩 THE PUZZLE: WHAT WE'RE SOLVING
─────────────────────────────────────────────────────────────────────────────

INPUT: A Sudoku puzzle with some numbers filled in

Example (easy.txt):
    5 3 _ _ 7 _ _ _ _
    6 _ _ 1 9 5 _ _ _
    _ 9 8 _ _ _ _ 6 _
    8 _ _ _ 6 _ _ _ 3
    4 _ _ 8 _ 3 _ _ 1
    7 _ _ _ 2 _ _ _ 6
    _ 6 _ _ _ _ 2 8 _
    _ _ _ 4 1 9 _ _ 5
    _ _ _ _ 8 _ _ 7 9

CHALLENGE: Fill 81 cells with numbers 1-9 so that:
    • Each row has 1-9 (no repeats)
    • Each column has 1-9 (no repeats)
    • Each 3×3 box has 1-9 (no repeats)

This is what we call a "Constraint Satisfaction Problem"

🔍 THE ALGORITHM: HOW WE SOLVE IT
─────────────────────────────────────────────────────────────────────────────

Your solver uses 5 techniques combined strategically:

TECHNIQUE 1: AC-3 CONSTRAINT PROPAGATION
─────────────────────────────────────────

What it does:
    • Start: Each empty cell could be 1, 2, 3, 4, 5, 6, 7, 8, or 9
    • Constraint: If row 0 already has a 5, no other cell in row 0 can be 5
    • Action: Remove 5 from all possibilities in row 0
    
How it works:
    1. Look at each GIVEN number (30 in easy puzzle)
    2. Remove that number from all cells that share:
       - Same row
       - Same column
       - Same 3×3 box
    3. Repeat until no more changes

Example:
    Cell (0,0) = 5 (given)
    
    This removes 5 from:
    • Row 0: cells (0,1), (0,2), ..., (0,8)
    • Col 0: cells (1,0), (2,0), ..., (8,0)
    • Box: cells (1,1), (1,2), (2,1), (2,2)
    
Result:
    • Before: Cell (0,2) could be {1,2,3,4,5,6,7,8,9}
    • After:  Cell (0,2) could be {1,2,3,4,6,7,8,9} (5 removed!)

Impact on Easy Puzzle:
    • 30 given clues create strong constraints
    • AC-3 alone determines ALL 81 cells!
    • BACKTRACK count: 0 (no guessing needed)


TECHNIQUE 2: FORWARD CHECKING
──────────────────────────────

What it does:
    • After assigning a value to a cell
    • Immediately reduce domains of affected neighbors
    • Check for domain wipeout (fail early)

Why it matters:
    • Detects conflicts immediately
    • Prevents exploring dead branches
    • Saves massive computation


TECHNIQUE 3: NAKED SINGLES
──────────────────────────

What it does:
    • Find cells that have only 1 possible value
    • Mark them as "forced" assignments
    • Assign them automatically
    • Propagate to neighbors (chain reaction)

Example:
    After AC-3, cell (0,3) = {4}  (only one possibility)
    
    Action:
    • Assign (0,3) = 4
    • Remove 4 from neighbors
    • This might make other cells forced
    • Continue until chain reaction stops

Impact:
    • Creates cascading assignments
    • Often solves puzzle without backtracking
    • Reduces search space dramatically


TECHNIQUE 4: MRV HEURISTIC (Minimum Remaining Values)
──────────────────────────────────────────────────────

When you MUST guess (backtracking phase):
    
    Problem: Multiple cells still have multiple possibilities
    Question: Which cell to guess on first?
    
    Naive approach:
    • Pick first empty cell: domain size 9
    • Try all 9 values: 9 branches
    • For each, explore rest: 8^80 possibilities!
    • Result: EXPONENTIAL EXPLOSION
    
    Smart approach (MRV):
    • Look at ALL empty cells
    • Find the one with SMALLEST domain
    • Example: Find cell with only 2 possibilities
    • Try 2 values instead of 9: 2 branches
    • Result: EXPONENTIAL REDUCTION!

Why it works:
    • Most constrained cells likely to fail if wrong
    • Detects contradictions immediately
    • Saves exploring huge branches

Example:
    Cell A: domain = {1, 2, 3, 4, 5, 6, 7, 8, 9}  (9 options)
    Cell B: domain = {3, 5}                         (2 options) ← MRV picks this!
    Cell C: domain = {1, 2, 4, 6, 7, 8, 9}         (7 options)
    
    Pick B first: Only 2 possibilities to try
    Smart branching saves hours of computation!


TECHNIQUE 5: BACKTRACKING SEARCH
─────────────────────────────────

When propagation & singles aren't enough (hard puzzles):

    Process:
    1. Use MRV to pick most constrained cell
    2. Try first value in its domain
    3. Apply forward checking
    4. If no contradiction: Recurse (continue)
    5. If contradiction: Skip this value
    6. Try next value
    7. If all values fail: Backtrack (undo)
    
    Stats collected:
    • BACKTRACK count: How many times we try a value
    • Failure count: How many lead nowhere
    
    Hard puzzle example:
    • Try 14 different values before finding solution
    • 7 of them lead to contradictions
    • BACKTRACK = 14, Failures = 7

USER INTERACTION (Browser):

Step 1: Select puzzle (radio button)
    ○ Easy (30 clues)
    ● Hard (18 clues)
    ○ Very Hard (18 clues)

Step 2: Click "Load Puzzle" button
    ↓ JavaScript: loadPuzzle()
    ├─ Prepare JSON: {puzzle: "hard"}
    ├─ Send: fetch('/load_puzzle', {POST, JSON})
    ↓
    SERVER RESPONDS:
    ├─ Read file: hard.txt
    ├─ Parse content
    ├─ Prepare JSON response:
    │  {
    │    grid: [[5,3,0,...], ...],
    │    clues: 18,
    │    difficulty: "Hard"
    │  }
    └─ Send to browser
    ↓
    BROWSER RECEIVES:
    ├─ Extract solution grid
    ├─ For each cell:
    │  ├─ Create div element
    │  ├─ Set text content
    │  ├─ Add CSS classes
    │  ├─ Add animation
    │  └─ Insert in DOM
    ├─ Apply fade-in animation
    └─ Grid visible with puzzle!

Step 3: Click "Solve" button
    ↓ JavaScript: solvePuzzle()
    ├─ Extract grid from DOM (81 cells)
    ├─ Prepare JSON: {grid: [[5,3,0,...], ...]}
    ├─ Send: fetch('/solve', {POST, JSON})
    ↓
    SERVER RUNS SOLVER:
    ├─ Receive grid
    ├─ Call solver.solve(grid)
    ├─ ALGORITHM EXECUTES (AC-3, etc.)
    ├─ Collect statistics
    ├─ Prepare JSON response:
    │  {
    │    solution: [[5,3,4,6,...], ...],
    │    stats: {
    │      backtrack_count: 14,
    │      failure_count: 7,
    │      difficulty: "Hard"
    │    }
    │  }
    └─ Send to browser
    ↓
    BROWSER DISPLAYS SOLUTION:
    ├─ Receive JSON response
    ├─ For each solved cell:
    │  ├─ Set text to solution value
    │  ├─ Add green color
    │  ├─ Add scale animation
    │  └─ Add fade-in effect
    ├─ Update statistics panel:
    │  ├─ BACKTRACK: 14
    │  ├─ Failures: 7
    │  ├─ Difficulty: Hard
    │  └─ Color codes for each
    └─ Beautiful, animated solution appears!
